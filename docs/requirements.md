# 要件定義 - Chain Maintenance App

## プロジェクトの目的

AWS上でRDS・HTTPS・Auto Scalingの各要素を実地検証するための学習台として、
バイクのチェーンメンテナンス記録Webアプリを構築する。

**本プロジェクトの主目的は技術検証であり、アプリ機能そのものではない。**
ただし「動くだけ」で終わらせず、ポートフォリオとして提示可能な完成度を目指す。

## 機能要件

### MVP(必須)

| ID | 機能 | 概要 |
|---|---|---|
| F-01 | バイク情報登録 | 車種名、前後スプロケ丁数、チェーンコマ数、タイヤ円周を登録 |
| F-02 | メンテ記録登録 | 日付、走行距離、使用ルブ、メモを登録 |
| F-03 | チェーン1周回転数計算 | 登録情報からホイール回転数を算出し表示 |
| F-04 | メンテ履歴一覧 | 過去のメンテ記録を時系列で表示 |

### 拡張機能(時間があれば)

| ID | 機能 | 概要 |
|---|---|---|
| F-11 | 次回メンテ推奨表示 | 前回メンテからの走行距離に基づく推奨 |
| F-12 | メンテ間隔グラフ | メンテ周期の推移を可視化 |

### スコープ外(意図的に実装しない)

- ユーザー認証(Cognito):学習スコープ外、自分専用前提
- マルチユーザー対応:同上
- モバイルアプリ化:レスポンシブHTMLで十分
- 車種マスタ機能:自分の1台で足りる

## 非機能要件

| 観点 | 目標値 | 備考 |
|---|---|---|
| 可用性 | 99%以下で可 | 学習用途、SLA不要 |
| レスポンスタイム | API応答 < 500ms | 四則演算のみなので余裕 |
| 同時接続数 | 10 req/s処理可能 | Auto Scaling検証時の負荷試験基準 |
| 月額コスト | 5,000円以下 | 学習完了後は停止してコストゼロ化 |
| セキュリティ | HTTPS必須、DB情報はSecrets Manager管理 | 平文DB接続情報はコードに書かない |

## 計算式仕様

### チェーン1周のホイール回転数
ホイール回転数 = チェーンコマ数 / (リアスプロケ丁数 × 2)

**根拠**:
- チェーン1コマ = 1/2インチ(ANSI規格)
- リアスプロケは1歯あたりチェーン1コマと噛む
- リアスプロケ1回転でチェーンは「丁数 × 2コマ」進む
- したがって、リアスプロケ(=リアホイール)が `チェーンコマ数 / (丁数 × 2)` 回転でチェーン1周

### 走行距離からの算出(拡張機能用)
タイヤ1回転の走行距離 = タイヤ円周(mm) / 1,000,000 (km)
チェーン1周の走行距離 = ホイール回転数 × タイヤ1回転の走行距離
