## 重要な注意点

VPC EndpointはNAT Gatewayの代替として「コスト削減」と言われがちだが、
**Interface型VPC Endpointは1個あたり月約1,000円+データ処理料**がかかる。
4個使うと約4,200円に達し、NAT Gateway(月5,000円)との差が小さい。

### コスト最適化の選択肢

#### 選択肢1: VPC Endpointを減らす
必須なのは以下のみに絞る:
- `ecr.dkr`, `ecr.api`(Interface):コンテナPull必須
- `s3`(Gateway、**無料**):ECRイメージレイヤー格納先
- `secretsmanager`(Interface):DB認証情報取得必須
- `logs`(Interface):CloudWatchログ出力

→ Interface型4個のまま、これ以上減らすと機能が壊れる。

#### 選択肢2: Fargate Sparkを使う(実験的)
Fargate Sparkはスポット的な価格だが、Auto Scaling検証には不向き。**却下**。

#### 選択肢3: 使わない時間帯は停止する(最有力)
- ECS Serviceのdesired count = 0 にするスクリプト
- CloudWatch EventsでのスケジュールOFF
- **実質コストを月3,000円程度まで圧縮可能**

### 推奨運用: 「使う時だけON」スクリプト戦略
[平日 9:00-22:00 だけ起動] の場合の月額試算:
稼働時間: 13h × 22日 = 286時間(通常720時間の約40%)
ALB: 2,500 × 0.4 = 約1,000円
ECS: 1,400 × 0.4 = 約560円
VPC Endpoint: 4,200 × 0.4 = 約1,680円
Aurora: 変わらず 約1,000円(ACU 0で待機)
その他固定費: 約400円
────────────────────────
合計: 約4,640円

停止スクリプトをPhase 4完了時に実装することを推奨。

## 無料クレジットの活用

### 失効前の対応チェックリスト

- [ ] 2026年7月24日: リソース棚卸し(2週間前)
- [ ] 2026年8月1日: 全リソース停止(1週間前)
- [ ] 2026年8月7日: 最終確認(失効当日)

**AWSのBillingアラートはクレジット失効「後」の課金のみ検知する**。
クレジット失効前にはアラームは発動しないため、カレンダー管理が必須。

## 負荷試験時の追加コスト

Phase 4で負荷試験を実施する際の一時的コスト増:
- ECSタスク数が1→4にスケールアウト → 実行時間数時間なら数十円
- Aurora ACUが0.2→1.0に急増 → 数時間で約50円
- ALB LCUが増加 → 数時間で約30円

**負荷試験1回あたり: 約100〜200円追加**。許容範囲。

## 学習完了後のコストゼロ化手順

4フェーズ完了後、継続コストを発生させないための完全停止手順:

1. ECS Service のdesired count を 0 に変更
2. Aurora Serverless v2 クラスタを**削除**(スナップショットは自動保持)
3. ALB を削除
4. VPC Endpoint をすべて削除
5. NAT Gateway があれば削除
6. CloudWatch Logs グループをエクスポート後削除

**残してもいいもの**:
- ECRイメージ(月数円)
- Route 53 ホストゾーン(月75円、後日再開する可能性があれば)
- ドメイン(年800円)

→ 停止後の月額: **約100円以下**

## コストアラート設定状況

CloudWatch Billing アラームで以下を設定済み(us-east-1):
- `billing-warning-20usd`: 月額$20(約3,000円)到達時にメール通知
- `billing-critical-53usd`: 月額$53(約8,000円)到達時に緊急通知

**注意**: 無料クレジット消費中はアラートが発動しない。
クレジット失効後、または予想外の課金時にのみ機能する設計。

## コストに関する注意

学習台プロジェクトで最も多い失敗パターンは、**「終わった後に停止し忘れて毎月課金される」**こと。
カレンダーに「Phase 4完了1週間後に全リソース棚卸し」を登録しておくこと。
